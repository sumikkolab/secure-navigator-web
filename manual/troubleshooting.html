Chapter 9: トラブルシューティング

よくある問題と対処方法

WebView2 Runtime が見つからない

症状: アプリ起動時に「WebView2 Runtime が見つかりません」というエラーが表示される。または PDF エクスポート時にエラーが発生する。

原因: SecureNavigator のレポートプレビューと PDF エクスポートには Microsoft Edge WebView2 Runtime が必要です。Windows 11 および最新の Windows 10 には通常プリインストールされていますが、一部の企業管理環境や古い環境では未インストールの場合があります。

対処方法:

  1. Microsoft 公式サイトから WebView2 Runtime の「Evergreen Bootstrapper」をダウンロードします。
  2. ダウンロードしたインストーラーを実行してインストールします。
  3. SecureNavigator を再起動します。

企業管理下の PC で自身でのインストールが制限されている場合は、IT 管理者に WebView2 Runtime の配布を依頼してください。

セッション上限に達した

症状: 新しい診断を作成しようとすると「セッション数の上限に達しています」と表示される。

原因: Free 版では1件、Pro 版では3件のセッション保存上限があります。

対処方法:

  • 不要になった既存のセッションをホーム画面から削除してから、新しい診断を作成してください。
  • より多くのセッションが必要な場合は、Pro 版へのアップグレードを検討してください。Pro 版では最大3件まで保存できます。

セッションを削除すると、そのセッションの回答データと結果はすべて失われます。削除前に必要なデータがないか確認してください。

利用規約の再同意を求められる

症状: アプリ起動時に、以前同意済みの利用規約への再同意を求められる。

原因: アプリの更新により利用規約の内容が変更された場合、新しいバージョンの規約への同意が必要になります。これは正常な動作です。

対処方法:

  1. 更新された利用規約の内容を確認します。
  2. 「同意する」をクリックして続行します。

既存の診断データへの影響はありません。同意後、通常通り利用を再開できます。

データの保存場所と初期化

データの保存場所: SecureNavigator のすべてのデータ（セッション、設定、利用規約の同意状態）は以下のフォルダに保存されます。

%LocalAppData%\SecureNavigator\

通常は C:\Users\[ユーザー名]\AppData\Local\SecureNavigator\ に該当します。

データの初期化: すべてのデータを初期状態に戻したい場合は、上記フォルダを削除してからアプリを再起動してください。利用規約への再同意が求められ、すべてのセッションと設定がリセットされます。

データフォルダの削除はすべての診断データの消失を意味します。この操作は取り消せません。必要に応じて、事前にフォルダのバックアップをとってください。

アプリが起動しない・クラッシュする

対処方法:

  1. OS 要件の確認: Windows 10 Version 2004 以降、または Windows 11 であることを確認してください。
  2. WebView2 Runtime の確認: 上記の手順で WebView2 Runtime がインストールされているか確認してください。
  3. アプリの修復: Windows の「設定 > アプリ > インストール済みアプリ」から SecureNavigator を見つけ、「詳細オプション」から「修復」を実行してください。
  4. 再インストール: 修復で解決しない場合は、アプリをアンインストールしてから Microsoft Store から再インストールしてください。

上記の方法で解決しない場合は、サポートページから OS バージョン、アプリバージョン、エラーメッセージを添えてお問い合わせください。

Chapter 9: Troubleshooting

Common issues and solutions

WebView2 Runtime Not Found

Symptom: An error "WebView2 Runtime not found" appears on startup, or an error occurs during PDF export.

Cause: SecureNavigator's report preview and PDF export features require the Microsoft Edge WebView2 Runtime. It is usually pre-installed on Windows 11 and recent Windows 10 builds, but may not be present in some enterprise-managed or older environments.

Solution:

  1. Download the "Evergreen Bootstrapper" from the Microsoft official site.
  2. Run the downloaded installer to install the runtime.
  3. Restart SecureNavigator.

If software installation is restricted on your enterprise-managed PC, ask your IT administrator to deploy the WebView2 Runtime.

Session Limit Reached

Symptom: A message "Session limit reached" appears when attempting to create a new assessment.

Cause: The Free edition allows 1 saved session, and the Pro edition allows up to 3.

Solution:

  • Delete an existing session you no longer need from the Home screen, then create a new assessment.
  • If you need more sessions, consider upgrading to the Pro edition, which supports up to 3 saved sessions.

Deleting a session permanently removes all of its response data and results. Verify that you no longer need the data before deleting.

Terms Re-acceptance Required

Symptom: The app asks you to accept terms of use that you have previously accepted.

Cause: When an app update includes changes to the terms of use, acceptance of the new version is required. This is expected behavior.

Solution:

  1. Review the updated terms of use.
  2. Click "Accept" to continue.

Your existing assessment data is not affected. After accepting, you can resume normal usage.

Data Location and Reset

Data location: All SecureNavigator data (sessions, settings, terms acceptance state) is stored in the following folder:

%LocalAppData%\SecureNavigator\

This typically resolves to C:\Users\[username]\AppData\Local\SecureNavigator\.

Data reset: To reset all data to the initial state, delete the above folder and restart the application. You will be asked to accept the terms of use again, and all sessions and settings will be reset.

Deleting the data folder permanently removes all assessment data. This action cannot be undone. Consider backing up the folder before proceeding if needed.

App Does Not Start or Crashes

Solution:

  1. Check OS requirements: Verify that you are running Windows 10 Version 2004 or later, or Windows 11.
  2. Check WebView2 Runtime: Follow the steps above to verify the WebView2 Runtime is installed.
  3. Repair the app: Go to Windows "Settings > Apps > Installed apps", find SecureNavigator, click "Advanced options", and run "Repair".
  4. Reinstall: If repair does not resolve the issue, uninstall the app and reinstall it from the Microsoft Store.

If none of the above resolves the issue, please contact us through the Support page with your OS version, app version, and error message.
